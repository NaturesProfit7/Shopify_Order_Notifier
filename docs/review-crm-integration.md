# Код-рев'ю: Інтеграція keyCRM — кнопка «Створити в CRM»

## Статус: ✅ Реалізовано

---

## Контекст і мета

Telegram-бот отримує замовлення Shopify і дозволяє менеджерам ними керувати. Після переведення замовлення у статус PAID менеджер може одним натисканням перенести його в CRM-систему keyCRM: автоматично створюється покупець (або прив'язується існуючий), а всі деталі замовлення передаються в коментар менеджера для ручного рознесення.

### Поведінка

- Кнопка `🏪 Створити в CRM` видна лише у замовлень зі статусом **PAID**
- Після успішного створення — кнопка змінюється на неактивну `✅ В CRM` (блокування дублів)
- Повторне натискання → popup `⚠️ Вже створено в CRM`
- Після створення в чат надсилається повідомлення зі посиланням на замовлення у CRM
- CRM ID зберігається в `order.raw_json["_crm_order_id"]` — без міграції БД
- Існуючий покупець: keyCRM автоматично прив'язує новий заказ до покупця з таким самим телефоном

### Правила форматування номерів з крапками

- Символ-розділювач: `•` (U+2022)
- Формат: `+38•XXX•XXX•XX•XX`
- Джерело: properties товарів з іменем `Номер телефону` або `Другий номер або коротенька фраза`
- Виявлення номера: якщо після видалення не-цифр залишається **9+ цифр** — вважається номером
- Телефон з `shipping_address` **не** форматується з крапками
- Виводиться кожен номер (без дедуплікації)
- Виводиться в кінці коментаря з заголовком `Телефони:` і підзаголовком товару
- Якщо в замовленні не знайдено жодного номера — секція `Телефони:` не виводиться

### Нормалізація перед форматуванням

### `app/services/keycrm_service.py`

**Статус:** ✅ Реалізовано

**Ключові функції:**
- `create_crm_order(order: Order) -> dict` — головна функція, викликається через `run_in_executor`
- `_format_manager_comment(raw_json: dict) -> str` — збирає повний текст коментаря
- `_build_phones_section(line_items: list) -> str` — секція телефонів внизу
- `_format_phone_dotted(phone_str: str) -> str` — форматування `+38•XXX•XXX•XX•XX`
- `_is_phone_number(value: str) -> bool` — 9+ цифр після видалення не-цифр
- `_get_phones_from_properties(properties: list) -> list[str]` — витягує телефони з properties

**Архітектурні рішення:**
- `requests.Session()` з Bearer-токеном (аналог `shopify_service.py`)
- Конфіг через `os.getenv()` на рівні модуля
- `source_uuid` = номер замовлення Shopify для зворотного зв'язку
- `buyer.email` = `None` якщо відсутній (API приймає nullable)

**Граничні випадки:**
- `raw_json` = None → `raw_json or {}` захист скрізь
- `shipping_lines` порожній → рядок "Доставка:" не виводиться
- Properties без `_` в імені → показуємо (стандартна фільтрація Shopify)
- `Другий номер або коротенька фраза` = фраза (< 9 цифр) → пропускаємо

### `app/bot/routers/management.py`

**Статус:** ✅ Реалізовано

**Потік виконання хендлера:**
1. `check_permission` → інакше мовчки виходимо
2. Дістати замовлення з БД
3. Перевірити `_crm_order_id` → якщо є: popup `⚠️ Вже створено в CRM`
4. `callback.answer("⏳ Створюю...")` — одразу, щоб не завис індикатор
5. `run_in_executor(None, create_crm_order, order)` — не блокує event loop
6. Зберегти: `order.raw_json = {**order.raw_json, "_crm_order_id": crm_id}` (повне переприсвоєння для JSONB)
7. `edit_reply_markup` — кнопка стає `✅ В CRM`
8. Відправити повідомлення зі посиланням у чат

**`session.expunge(order)`** перед `run_in_executor` — відв'язує ORM-об'єкт від сесії. SQLAlchemy session не є thread-safe; передача прив'язаного об'єкта в потік небезпечна. Після expunge scalar-атрибути (raw_json, phone, name) зберігаються, але об'єкт більше не звертається до сесії.

**Два окремих `get_session()`** — перший для читання/перевірки + expunge, другий для запису після HTTP-запиту. Сесія не тримається відкритою під зовнішній запит.

**`await`-виклики поза `with get_session()`** — Telegram API-виклики (`edit_reply_markup`, `send_message`) відбуваються після закриття сесії. Клавіатура будується всередині сесії (синхронно) і зберігається в локальній змінній.

**Локальний імпорт** `from .orders import get_correct_keyboard` всередині функції — усталений паттерн у management.py для уникнення циклічних імпортів.

## Як перевірити вручну

1. Знайти замовлення зі статусом PAID → переконатися що з'явилася кнопка `🏪 Створити в CRM`
2. Натиснути → у чаті має з'явитися повідомлення з посиланням `https://timosh-design.keycrm.app/app/orders/view/{id}`
3. Відкрити посилання → перевірити: покупець заповнений (ім'я, телефон, email), коментар містить всі деталі + секцію `Телефони:` внизу
4. Переконатися що кнопка змінилась на `✅ В CRM`
5. Натиснути `✅ В CRM` ще раз → popup `⚠️ Вже створено в CRM`
6. Перевірити замовлення з товарами що мають property `Номер телефону` → відповідний номер у форматі `+38•XXX•XXX•XX•XX` має бути внизу коментаря
7. Перевірити замовлення де `Другий номер або коротенька фраза` = фраза (не номер) → секція телефону для цього поля відсутня
8. Зробити повторний заказ того ж клієнта → новий заказ у CRM прив'язаний до існуючого покупця
